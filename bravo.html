Speaker Name: Simon Feriancek

Presentation Title: Reference Software or, “How Can I shoehorn Bitcoin into another presentation”

Description: A brief explanation of what Reference Software is and why it’s a thing. Using Bitcoin and Bittorrent to elaborate points, as well as provide something pretty to look at.

Key Points:

  • Reference Software is aimed at a technical audience, and not intended for widespread public use. Intended use is as a complete implementation example
  • Visual Style is kept to a minimum; feature completeness trumps usability
  • Given enough time, Reference Software will evolve and improve usability
  • Ideally the Reference is superseded by other software that targets specific groups of users
